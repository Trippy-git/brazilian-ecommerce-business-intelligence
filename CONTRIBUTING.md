# Contributing

First, thank you for taking the time to contribute to this project!

This repository was created as an end-to-end Business Intelligence case study using the Brazilian E-Commerce (Olist) dataset. Contributions that improve code quality, documentation, dashboard usability, or analytical insights are always welcome.

---

# Ways to Contribute

You can contribute by:

- Improving SQL query performance
- Adding new analytical modules
- Enhancing Power BI dashboards
- Improving documentation
- Fixing bugs
- Optimizing DAX measures
- Suggesting new business insights

---

# Development Workflow

## 1. Fork the Repository

Create your own copy of the repository.

---

## 2. Clone

```bash
git clone https://github.com/Trippy-git/brazilian-ecommerce-business-intelligence.git
```

---

## 3. Create a Branch

```bash
git checkout -b feature/my-feature
```

---

## 4. Commit Changes

```bash
git commit -m "Add meaningful description"
```

---

## 5. Push

```bash
git push origin feature/my-feature
```

---

## 6. Open a Pull Request

Describe:

- What changed
- Why it changed
- Screenshots (if dashboards changed)

---

# Coding Standards

## SQL

- Use uppercase SQL keywords.
- Format queries consistently.
- Prefer CTEs over deeply nested subqueries.
- Use meaningful aliases.
- Comment complex logic.

---

## Power BI

- Keep dashboard layouts consistent.
- Reuse DAX measures whenever possible.
- Minimize unnecessary visuals.
- Maintain consistent color palettes.

---

## Documentation

Use Markdown headings consistently.

Keep documentation business-focused and technically accurate.

---

# Reporting Issues

If you discover a bug:

- Describe the issue
- Include screenshots if applicable
- Provide reproduction steps
- Suggest possible improvements

---

# Thank You

Your contributions help improve this repository and make it a more valuable learning resource for the Business Intelligence community.
